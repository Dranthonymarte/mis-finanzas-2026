Visión general del sistema

PWA monolítica de gestión financiera personal para Anthony & Isabel Marte en Venezuela. Toda la app vive en un único index.html de ~700 KB con 12,623 líneas de HTML, CSS y JavaScript vanilla.

🔴 calcFire() NO suma localStorage
Usa ONLY EXCEL_DATA[m].ahorros — sumar finanzas_cochinito_v2 de localStorage causaría doble conteo (resuelto en FIX-12).
🔴 Texto de IA → renderIAText() antes de innerHTML
Todo response de Groq DEBE pasar por renderIAText() o los caracteres ~, ** aparecen en pantalla.
🔴 CUENTAS[].moneda = "USD" | "BS" únicamente
El valor "internacional" NO existe en el schema. Usarlo en condicionales causa bugs en transferencias (FIX-17).
🔴 modal-cochinito NO existe
El cochinito es una card del dashboard, no un modal overlay. No hay id="modal-cochinito".
🟡 CACHE_VERSION → incrementar en cada deploy
service-worker.js: finanzas-v16 → v16 → v17... Sin este cambio el navegador sirve código viejo en caché.
🟡 const sb = supabase.createClient() — primera línea del bloque #3
No mover, no envolver en async, no condicionar. Si el CDN falló, el onerror del script tag activa el fallback a unpkg.
🟡 showCardInfo → solo una versión unificada
Nunca duplicar — la primera delega a _showCardInfoV2 para tipos dinámicos (FIX-4 Batch-D).
🔵 openNuevaCuenta() cierra overlays antes de abrir
Siempre llamar el bloque de cierre de overlays al inicio para evitar modales superpuestos (FIX-16).
